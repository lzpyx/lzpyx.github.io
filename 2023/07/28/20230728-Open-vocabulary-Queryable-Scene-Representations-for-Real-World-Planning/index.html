affordance grounding

Motivation

prior attempts to apply LLMs to real-world robotic tasks are limited by the lack of grounding in the surrounding scene

NLMap serves as a framework to gather and integrate contextual information into LLM planners, allowing them to see and query available objects in the scene before generating a context-conditioned plan.

  • ObjectNavйҮҚеӨҚжҺўзҙўзҡ„жЁЎејҸжҳҜдҪҺж•Ҳзҡ„

    Though a robot can avoid building a semantic representation by finding objects each time they are required, e.g., with Object Goal Navigation, this repeated exploration can be inefficient

  • Persistent Scene RepresentationеҸҜд»ҘйҒҝе…ҚйҮҚеӨҚзҡ„жҺўзҙўпјҢдҪҶд»ҘеҫҖе·ҘдҪңд»…иғҪе®ҡдҪҚжһ„еӣҫйҳ¶ж®өйў„е®ҡд№үзҡ„зӣ®ж Үзұ»еҲ«пјҢж— жі•е®һзҺ°ејҖж”ҫиҜҚжұҮ

  • еҰӮдҪ•иҝһжҺҘsemantic scene representationе’Ңplanning
    • дҪҝз”ЁLLMжңҖеӨ§зҡ„й—®йўҳеңЁдәҺпјҢLLMs are not grounded in the physical world

Our work differs fromObject Goal Navigation since the eventual goal is not purely finding objects, but using object presence and location information for planning.

Our work can use the representation from a single exploration for many downstream planning tasks without the need to run Object Goal Navigation every time.

йҰ–е…Ҳи®©жңәеҷЁдәәжҺўзҙўеңәжҷҜе№¶ж”¶йӣҶи§ӮеҜҹз»“жһңгҖӮдёҺзұ»еҲ«ж— е…ізҡ„еҢәеҹҹжҸҗи®®зҪ‘з»ңжҸҗеҮәж„ҹе…ҙи¶Јзҡ„еҢәеҹҹе№¶еҲ©з”ЁVLMsжҸҗеҸ–зү№еҫҒгҖӮзү№еҫҒе’ҢеҜ№иұЎиҫ№з•ҢжЎҶйҖҡиҝҮеӨҡи§ҶеӣҫиһҚеҗҲз®—жі•иҒҡеҗҲпјҢд»ҘиҺ·еҫ—open-vocabulary scene representations

еҪ“дәәзұ»еҸ‘еҮәжҢҮд»Өж—¶пјҢLLMsдјҡе°ҶжҢҮд»Өи§Јжһҗдёәзӣёе…іеҜ№иұЎзҡ„еҲ—иЎЁгҖӮз„¶еҗҺжҹҘиҜўеңәжҷҜиЎЁзӨәд»ҘдәҶи§ЈиҝҷдәӣеҜ№иұЎзҡ„еҸҜз”ЁжҖ§е’ҢдҪҚзҪ®гҖӮеҸҜжү§иЎҢйҖүйЎ№жҳҜж №жҚ®жүҫеҲ°зҡ„еҶ…е®№з”ҹжҲҗзҡ„гҖӮжңҖеҗҺпјҢжңәеҷЁдәәж №жҚ®жҢҮд»Өе’ҢжүҫеҲ°зҡ„зү©дҪ“и§„еҲ’е№¶е®ҢжҲҗд»»еҠЎгҖӮ

After exploring the scene, when a human gives the robot an instruction, the robot will propose potentially involved objects in the scene and query the gathered scene representation for their locations and availability

Scene Representation

еңЁжҺўзҙўжңҹй—ҙпјҢNLMapеңЁжүҖжңүи§ӮеҜҹеҲ°зҡ„RGBеӣҫеғҸдёҠдҪҝз”Ёзұ»еҲ«ж— е…ізҡ„еҢәеҹҹе»әи®®зҪ‘з»ңпјҲViLDдёӯзҡ„пјүжқҘеҫ—еҲ°зҡ„жҪңеңЁеҜ№иұЎеҢәеҹҹ$I$пјҢеҲ©з”ЁViLDе’ҢCLIPжҸҗеҸ–еҢәеҹҹзү№еҫҒпјҢдҪҝз”ЁдёҖдёӘдёүе…ғз»„(зү№еҫҒпјҢдҪҚзҪ®пјҢе°әеҜё)жқҘжҸҸиҝ°дёҖдёӘеҜ№иұЎпјҡ

$c_i=(\phi_i,p_i,r_i)$

е…¶дёӯ$\phi_i=[\Phi_{clip_img}(I_i),\Phi_{vild_img}(I_i)]$ ,еңәжҷҜдёӯзҡ„жүҖжңүеҜ№иұЎдёүе…ғз»„жһ„жҲҗдәҶеңәжҷҜеҶ…е®№$\mathcal{C}$

Querying the Representation

дёәдәҶе®ҢжҲҗз”ұдәәзұ»жҢҮд»ӨжҢҮе®ҡзҡ„д»»еҠЎпјҢжңәеҷЁдәәйңҖиҰҒжҹҘиҜўеңәжҷҜиЎЁзӨәжқҘиҺ·еҫ—зӣёе…ідҝЎжҒҜ

    • дҪҝз”Ёfew-shot promptеҜ№LLMиҝӣиЎҢжҸҗзӨәпјҢеҲ©з”ЁLLMжҸҗеҸ–еҮәе®ҢжҲҗжҢҮд»ӨжүҖйңҖиҰҒеҜ»жүҫзҡ„еҜ№иұЎ

  • з„¶еҗҺж №жҚ®еңЁеңәжҷҜдёӯжүҫеҲ°зҡ„еҜ№иұЎз”ҹжҲҗеҸҜжү§иЎҢйҖүйЎ№пјҢз„¶еҗҺжҢүз…§instructionиҝӣиЎҢи®ЎеҲ’е’Ңжү§иЎҢ

  • жҹҘиҜўж—¶еҗҢж—¶дҪҝз”ЁCLIP embeddingе’ҢViLD embeddingпјҲеҒҡensemble,е°ұжҳҜдҪҝз”ЁзӣёдјјеәҰжӣҙй«ҳзҡ„пјүпјҢеӣ дёәеүҚиҖ…иғҪжӣҙеҘҪең°жЈҖжөӢеҲҶеёғеӨ–еҜ№иұЎпјҢиҖҢеҗҺиҖ…еҜ№еёёи§ҒеҜ№иұЎжӣҙйІҒжЈ’гҖӮд»ҘдёӢжҳҜзӣёдјјеәҰзҹ©йҳө$D$зҡ„и®Ўз®—иҝҮзЁӢпјҡ

    • the top k nearest neighbor elements for object name $y_i$ can be found in the scene representation $\mathcal{C}$,ж №жҚ®зӣёдјјеәҰзҹ©йҳөи®ҫзҪ®йҳҲеҖјиҝҮж»ӨжҺүдҪҺзҪ®дҝЎеәҰзҡ„зӣ®ж Ү

ејҖж”ҫиҜҚжұҮзҡ„еңәжҷҜиЎЁзӨәеҸҜд»ҘдёҺLLMs-based Plannerз»“еҗҲдҪҝз”ЁпјҢдҪҝжңәеҷЁдәәиғҪеӨҹеңЁзңҹжӯЈдёҚеҸ—жҺ§еҲ¶зҡ„зҺҜеўғдёӯж“ҚдҪң
